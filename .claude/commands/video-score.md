# video-pipeline 质量评分系统

对 `video-pipeline` 生成的产出物进行多维度质量评分，帮助持续提升内容质量。

## 使用方式

- `/video-score latest` — 自动找最近一次生成的 article.md 评分
- `/video-score /path/to/article.md` — 对指定图文文章评分
- `/video-score /path/to/video.mp4` — 对指定视频评分（需同目录有 article.md）

---

## 评分维度总览

| 维度 | 满分 | 说明 |
|------|------|------|
| A. 素材质量 | 54分 | 每条新闻最高9分，共6条 |
| B. 内容质量 | 30分 | 真实性、深度、来源 |
| C. 格式规范 | 10分 | 结构完整性、写作规范 |
| D. 视频质量 | 6分 | 仅评 video.mp4 时生效 |
| **总分** | **100分** | 加权后标准化至100分 |

---

## A. 素材质量评分（满分 54 分 → 标准化至 40 分）

逐条检查每条新闻的 `![]()` 素材行：

### 素材类型评分表

| 素材类型 | 原始分 | 判断标准 |
|---------|--------|---------|
| 🎬 视频（mp4/webm/YouTube） | **9分** | `![...](...mp4)` 或 YouTube/Bilibili iframe |
| 🌀 动态图（GIF/APNG） | **6分** | URL 含 `.gif` 或 `animated` |
| 🖼️ 截图/预览图（静态） | **3分** | Playwright 截图、og:image、README 内嵌图 |
| ➖ 无素材 | **0分** | 缺少 `![]()` 行 |
| ❌ 虚假/不匹配 | **-10分** | URL 404、与新闻内容无关、编造图片 |

### 验证规则
- **URL 合法性**：对每个图片 URL 执行 HEAD 请求，4xx/5xx → -10分
- **内容匹配性**：图片 alt 文字须与新闻主题相关，明显不符 → -10分
- **本地路径**：检查文件是否存在，不存在 → -10分

### 输出示例
```
A. 素材质量
  #1 oh-my-codex     [截图]  3/9 ✅
  #2 openscreen      [预览图] 3/9 ✅
  #3 oh-my-claudecode [截图] 3/9 ✅
  #4 hermes-agent    [banner] 3/9 ✅
  #5 Railway         [截图]  3/9 ✅
  #6 VeraCrypt       [截图]  3/9 ✅
  原始得分：18/54
  标准化：13.3/40 分
  提升建议：寻找 README 中的演示 GIF（目标 6分/条）
```

## B. 内容质量评分（满分 30 分）

### B1. 新闻真实性（10分）
用 Playwright 抽查 **2条** 新闻的来源 URL，验证：
- URL 可访问（非 404）：+3分/条，共 +6分
- 标题与页面内容匹配：+2分/条，共 +4分

### B2. 摘要质量（8分）
检查每条新闻的 blockquote 摘要：
- 每条摘要字数在 80-170 字之间：满足则 +1分，最多 +6分
- 全文摘要总字数 ≥ 400 字：+2分

### B3. 要点卡片完整性（6分）
- 每条新闻有 5 个要点卡片（`- 📈/💡/⚡/🔧/📊`）：每条 +1分，最多 +6分

### B4. 禁用词检测（6分，违规扣分）
- 基准 6分，检测全文出现以下词汇各扣 1分（最多扣完）：
  `惊艳` `颠覆` `革命性` `史诗级` `遥遥领先` `大家好` `感谢阅读` `我们认为` `你认为`

### 输出示例
```
B. 内容质量
  B1 真实性：抽查 #2(openscreen) URL 可访问 ✅ 标题匹配 ✅ +5分
             抽查 #5(Railway) URL 可访问 ✅ 标题匹配 ✅ +5分
  B2 摘要质量：6条均在字数范围内 ✅ 总字数≥400 ✅  8/8
  B3 要点卡片：6条 × 5个 = 30个，全部存在 ✅      6/6
  B4 禁用词：未检测到违规词汇 ✅                   6/6
  内容质量得分：24/30
```

## C. 格式规范评分（满分 10 分）

### C1. Frontmatter 完整性（3分）
必须包含：`title` `type` `date` `channel_color` `tab_label` `tags`，每缺一项 -0.5分

### C2. 标题格式（2分）
- 文章标题符合 `关键事件；关键事件【XX 周报 YYYY-MM-DD】` 格式：+2分

### C3. 结构完整性（3分）
- 有 H1 日期标题：+0.5分
- 有 `## 概览` 区块：+0.5分
- 每条新闻末尾有来源 URL 代码块：+1分
- 结尾有固定两行（提示/作者）：+1分

### C4. 写作规范（2分）
- 来源归因开头（`近日，`/`据媒体报道，` 等）覆盖率 ≥ 80%：+1分
- 数字使用阿拉伯数字（抽查3处）：+1分

### 输出示例
```
C. 格式规范
  C1 Frontmatter：6项齐全 ✅              3/3
  C2 标题格式：符合规范 ✅                2/2
  C3 结构完整：H1✅ 概览✅ 来源URL✅ 结尾✅  3/3
  C4 写作规范：来源归因100% ✅ 数字规范✅   2/2
  格式规范得分：10/10
```

## D. 视频质量评分（满分 6 分，仅评 video.mp4）

用 `ffprobe` 检测：

| 指标 | 满分 | 判断标准 |
|------|------|---------|
| 分辨率 | 2分 | ≥1920×1080: 2分 / ≥1280×720: 1分 |
| 时长合理 | 2分 | 60-900秒: 2分 / 30-60秒或>900秒: 1分 |
| 有音频流 | 1分 | 存在 AAC 音频流 |
| 文件大小 | 1分 | 1MB-500MB 范围内 |

---

## 综合评分计算

```
原始分 = A原始分/54×40 + B得分 + C得分 + D得分（可选）
总分（满100）= min(原始分, 100)
```

### 评级标准

| 总分 | 评级 | 建议 |
|------|------|------|
| 90-100 | ⭐⭐⭐⭐⭐ S 级 | 可直接发布 |
| 75-89  | ⭐⭐⭐⭐ A 级 | 建议小幅优化后发布 |
| 60-74  | ⭐⭐⭐ B 级 | 需优化素材或内容后再发布 |
| 40-59  | ⭐⭐ C 级 | 建议重新生成 |
| < 40   | ⭐ D 级 | 质量不达标，请重新运行流程 |

## 执行流程

**S1 — 定位文件**
- `latest`：查找 `/root/video-pipeline/output/` 下最新日期目录的 article.md
- 指定路径：直接读取

**S2 — 读取并解析 article.md**
- 解析 frontmatter
- 提取所有新闻条目（H2 标题、摘要、要点卡片、素材行、来源 URL）

**S3 — 执行各维度评分**
- A：验证每条素材（URL HEAD 检查 + 类型判断）
- B：Playwright 抽查2条来源 + 字数统计 + 要点计数 + 禁用词扫描
- C：结构检查（纯文本解析）
- D：若存在 video.mp4，运行 ffprobe

**S4 — 输出评分报告**

```
╔══════════════════════════════════════╗
║     video-pipeline 质量评分报告      ║
╚══════════════════════════════════════╝

文件：/root/video-pipeline/output/2026-04-08/github/article.md
类型：github | 日期：2026-04-08 | 新闻数：6

┌─────────────────┬──────┬──────┬──────┐
│ 维度            │ 得分 │ 满分 │ 评价 │
├─────────────────┼──────┼──────┼──────┤
│ A 素材质量      │ 13.3 │  40  │  C   │
│ B 内容质量      │ 24.0 │  30  │  A   │
│ C 格式规范      │ 10.0 │  10  │  S   │
│ D 视频质量      │  —   │   6  │  —   │
├─────────────────┼──────┼──────┼──────┤
│ 综合总分        │ 47.3 │ 100  │  C   │
└─────────────────┴──────┴──────┴──────┘

评级：⭐⭐ C 级

💡 提升建议（按优先级）：
1. [素材] 6条新闻均为静态截图(3分)，寻找演示GIF可提升至6分 → 预计+13分
2. [素材] 检查各 repo 官网/文档站是否有视频演示 → 若有则+9分/条
3. [内容] 已达标，继续保持

下次运行目标：B 级（60分以上）
```

**S5 — 可选：生成改进建议清单**
针对每个失分点，给出具体可执行的改进步骤。
